Research Objective

The main objective of this research is to build an event-based system which can detect malicious URLs early in the attack lifecycle. In doing so, we want to increase accuracy of the Malicious URL Classifier by using ensemble modeling and achieve an higher perfomance using feature reduction. Scalability of the model and system will be build from the core to handle various volumes of URLs.

Individual Components

IT18071412 - S.W. Sajeth Jonathan

Individual research questions


1. Which ensemble model will give adequate accuracy without trading too much in perfomance? ​
2. How to collect training and testing data effectively?​
3. What visualisations would the user require to get a better understanding of the URLs found based on the event?

Individual Objectives


1. Experiment with different ML and DL models to achieve an acceptable ensemble model. Top priority will be given to experiment with a Bidirectional LSTM + CNN model.
2. Building a web user interface using Python Flask Framework which will allow users to export malicious URLs list in CSV, JSON and XML format​
3. Collect Training and Testing data from publicly available feeds and label the URLs with keywords. Data can be collected from these sources, - OpenPhish - PhishTank - Common Crawler - WhoDNS - Yandex Search API

IT18032666 - Wishvajith B.L.D.V.

Individual research questions


1. How to populate related keywords from the event given by the user? ​
2. How to match the populated keywords with the malicious URLs database?

Individual Objectives


1. Identify NLP strategies to populate related keyword token from the user-input event keyword.
2. Identifying a model to find related keywords for an event by aggregating with online sources.
3. Co-relate the populated keyword tokens with the phishing URLs database

IT18021912 - Ramanayaka A. M.

Individual research questions


How can the feature reduction be done to build an ensemble model that will be efficient enough to detect phishing URLs with better scalability and speed?​

Individual Objectives


1. URL Feature extraction will be one of the objectives that will help us find specific features related to malicious URLs which then can be used to train the model for better detection of malicious URLs.
2. URL Feature Analysis will be another objective which will be carried out after the feature extraction is done.
3. URL Feature Classification will then be done after the analysis to understand how many features that we were able to find commonly among the malicious URLs.
4. URL Feature Reduction can then be done to improve the efficiency of the system which will reduce the features that were extracted and classified.
5. Scenario Categorization will be another objective to fulfill into finding out the differences that it would cause for the features as well.

IT18034400 - Renu Harshatha A.

Individual research questions


1. How is our system going to scale depending on the number of URLs to scan?
2. Can a browser extension be added to endpoints to protect end users from accessing malicious web pages?
3. What logs and reports would the administrator need from the data collected from the broswer extension?

Individual Objectives


1. Identify a solution for scalability which can be compatible with Ensemble Model. Compare available solutions for scalability, in order to do this a deep analysis on how each model works need to be studied and their algorithms needs to be compared.
2. Integrate the chosen scalable solution (Nomad) to scale the proposed system in this research.
3. Improving the scalability of the system using both machine learning and deep learning techniques to enhance the efficiency and reliability of the system.
4. Develop a browser extension for the proposed solution, which analyse, detects and extracts malicious URLs real time.
